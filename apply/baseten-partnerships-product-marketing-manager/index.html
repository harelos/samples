Spec strategy teardown

Where you are today

Baseten is a performance-first AI inference platform ('fastest model runtimes, cross-cloud high availability'). Good social proof with quantified outcomes (3x speed, sub-300ms transcription) and startup logos (Zed, Wispr, Writer). Clear modality organization; multi-cloud/self-hosted/hybrid.

The gaps I'd tackle first

  • Hero messaging ('Inference is everything') is abstract and assumes baseline expertise, weak for evaluators not already deep in ML infra
  • Pricing is opaque: no visible structure, comparison, or ROI framing, so prospects can't self-qualify
  • Feature sprawl across ~6 offerings with no use-case-to-entry-point mapping creates decision paralysis
  • CTA hierarchy is flat, multiple undifferentiated 'Learn more' / 'Talk to an engineer' links dilute intent
  • Customer proof skews startup; thin enterprise case studies leave larger buyers uncertain

How I'd fix it

  • Add comparative positioning vs AWS SageMaker/Lambda and other inference providers so the performance claim has a reference point
  • Introduce use-case-based entry guidance ('If you're doing X, start here') plus transparent starter-tier pricing/ROI calculator to cut evaluation friction
  • Build enterprise proof (security, scale, migration case studies) and a differentiated primary CTA to move mid-funnel buyers
